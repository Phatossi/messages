$Page マクロコントロールを使用する
$TargetName Using Macro Controls
$ListInTocForProduct 00

$0のラックの最もパワフルな機能のひとつに、｢マクロコントロール｣があります。 ラックには、それぞれ8つのノブがついています。各ノブはユーザ定義が可能です。 各ノブでは、ラック内に置かれたデバイスの任意数のパラメータを同時にコントロールすることができます。 さらに、それぞれ独自のコントロール範囲や極性を設定できます。 もちろん、マクロノブはハードウェアMIDIコントローラを使ってもコントロールできるため、演奏での柔軟性が高まります。

このレッスンでは、マクロコントロールのアサインを作成・変更したり、MIDI経由でコントロールする方法について学びます。

Divider.tif
*このレッスンでは、Liveセットを使用し、実際に操作を行いながら学ぶことができます。準備が整ったら、 $Link ここ <Set:/Sets/Using Macro Controls.als> をクリックしてLiveセットをロードしましょう。*
Divider.tif


セットがロードされたら、下の[次のページ]ボタンをクリックして進みます。

あるパラメータをマクロノブにアサインするには、まず正しいデバイスチェーンが表示されていることを確認します。 まず[Air]チェーンをクリックして選択し、この操作を行いましょう。

-> アサインされたパラメータには、パラメータの左上隅に色のついた四角形と、アサイン先のマクロコントロールの名前を表示するテキストフィールドが表示されます。

$Page 複数のマクロコントロールをアサインする
$TargetName Multiple Macro Assignments

フィルタレゾナンスを同じマクロコントロールにアサインしてみましょう。 これを行うには、[Res]ノブをクリックして選択してから1番目のマクロのマップボタンをクリックし、アサインを完了します。
ResKnob.tif

1つのマクロを複数のパラメータにアサインしたので、ノブの名前は一般名[Macro 1]に戻ります。 これを、覚えやすい名前（[Filter]など）に変更しましょう。 マクロコントロールの名前を変更するには、コントロールにマウスオーバーして *$ContextMenu* を使用するか、コントロールをクリックして選択してから *$CommandMod-R* を使って[名称変更]コマンドを実行します。 アサインされたパラメータとマッピングブラウザのテキストフィールドの名前が更新されます。 これにより、何が何へアサインされているかが簡単に把握することができます。

これまでに、1つのデバイスの2つのパラメータを1つのマクロコントロールへアサインしました。 マクロコントロールは、同一のラック内に含まれている複数のデバイスにアサインすることもできます。 [Wood]チェーンのデバイスのフィルタパラメータをアサインするには、チェーンリスト内のエントリをクリックして選択します。
WoodChain.tif

その後、[Air]チェーン内の[Simpler]での操作同様、[Freq]および[Res]ノブをフィルタマクロにマッピングします。

完了したら、マッピングブラウザ内に4つのエントリが表示されます。
FourAssignments.tif

$Page マッピングブラウザ
$TargetName The Mapping Browser

マッピングブラウザでは、マッピングしたアサインの設定すべてを確認し、必要に応じて範囲を調整することができます。 デフォルトでは、ブラウザには以下の列が表示されます。

-> *Macro* - アサインされているマクロの各ノブの名前です。
-> *Path* - チェーンの名前です。名前の後にそのチェーン内にあるアサインされたデバイスの名前が示されています。
-> *Name* - アサインされているパラメータの名前です。
-> *Min* - マクロを逆時計回りに完全に回した場合のパラメータの値です。
-> *Max* - マクロを時計回りに完全に回した場合のパラメータの値です。

ブラウザのヘッダ部分で$ContextMenuすると、これらの列の表示と非表示を切り替えることができます。
BrowserContext.tif

多数のマッピングを扱う場合、特定の基準（例：[Macro 1]にアサインされているすべてのパラメータ）または特定のデバイスのすべてのパラメータなどにのみ焦点を合わせると便利です。 ヘッダ部分をクリックして、Macro（マクロ）・Path（パス）・Name（名前）でマッピングブラウザ内で並べ替えることができます。
SortingTriangle.tif

$Page マクロマッピングを編集する
$TargetName Editing Macro Mappings

マッピングブラウザの[Min]（最低）および[Max]（最高）フィールドを編集すると、マクロノブからマップされているパラメータへ伝えられる値の範囲を限定（または反転）することができます。 たとえば、[Air]の最低値を150Hzに限定するには、マッピングブラウザ内のエントリを選択します。
AirFilterSelect.tif

その後、その[Min]フィールドをクリックし、[150]を入力します。
TypedMinValue.tif

$0の他の数値フィールド同様、これらのテキスト部分はスライダーにもなっています。 マウスを使ってドラッグし、値を設定することもできます。

マクロノブを回すと、この1つの値だけが変更されます。 アサインされた他のパラメータは影響されません。

このフィルタの[Res]コントロールの[Max]値を[4]へ制限し、レゾナンス設定が非常に高い場合に生じるとげとげしいピークを抑えます。 [Air]フィルタフィールドの表示が以下のようになります。
AirFilterSettings.tif

マッピングブラウザの[Min]および[Max]フィールドは、パラメータで設定可能な値の範囲内でそれぞれ設定することができるため、｢反転｣するマッピングを簡単に作成することができます。 [Min]の値を[Max]の値より高く設定するだけです。 さらに簡単な方法では、マッピングブラウザのエントリを$ContextMenuすることで実行される[レンジを反転]コマンドを使用することができます。 この動作を行うと、[Min]と[Max]の値がすぐさま入れ替わります。 それでは、[Air]フィルタ周波数のアサインを使ってこの動作をやってみましょう。
InvertRange.tif

ここで、フィルタマクロを時計回りに回すと、フィルタカットオフが下がり、レゾナンスが上がります。

$Page マクロのMIDIコントロール
$TargetName MIDI Control of Macros

Divider.tif
ご使用のMIDIキーボードにノブやフェーダが付いていない場合、このレッスンのために別のコントロールサーフェスを設定する必要があります。 設定に関して、詳しくは $Link MIDIコントローラを接続する <Lesson:Builtin/3-Hardware Setup Category/3-Connecting MIDI Controllers> レッスンをご参照ください。
Divider.tif

マクロノブをコントロールするようMIDIノブとフェーダをアサインし、さらに柔軟な操作を行うことができます。 こういったMIDIアサインの作成方法は、$0での他のMIDIアサイン作成方法と同様です。 まず、｢MIDIマップモードスイッチ｣をクリックし（または$CommandMod-Mを押し）ます。
MIDIMapButton.tif

その後、MIDIマッピングするフィルタマクロをクリックして選択します。
MIDIKnob.tif

その後、アサインするハードウェアのノブ/フェーダを動かします。 MIDIマッピングモードを終了し、新しいコントローラアサインを使用します。

*ご注意ください* - MIDIマッピングモードの場合、マッピングブラウザにMIDIマッピングアサインが表示されます（マクロアサインとは異なります）。 MIDIブラウザとマクロブラウザは、[Min]および[Max]のアサイン範囲を限定したり反転する機能など、多くの機能を共有しています。 MIDIアサインの反転動作は、マクロアサインの反転動作と同じです。

しかし、特定のパラメータに対する反転マッピングが｢あらかじめ｣あるマクロのMIDIアサインを反転すると、そのパラメータのMIDIコントロールは、反転していないかのように動作します。 このような状態は、混乱の元になります。 マッピングは、すぐに分かりにくくなります。 パワフルな機能を使用するには、負担がつきものですのでご注意ください。

-> ラック内の1つまたは複数のパラメータにマクロコントロールをアサインする方法を学びました。 ラックを構築する際、マクロコントロールを慎重に設定することにより、複雑なインストゥルメントまたはエフェクトの構成を、変更の簡単なコントロールインターフェースの背後に隠すことができます。 マクロアサインを追加したり変更したりして、ワークフローを使って練習してみましょう。

-> 極端なサウンドデザインを行うには、マクロコントロールを1つのデバイス内の全く関係のない複数のパラメータにアサインしてみましょう。 たとえば、LFO周波数、フィルタタイプ、移調をコントロールする1つのマクロを使うと、パッチがめちゃめちゃになります。特に、[Min]および[Max]の値を変更すると大混乱を引き起こします。
