$Page オーディオ入力/出力の設定
$TargetName Setting Up Audio I/O
$Comment Windows Version

このレッスンでは、$0のオーディオI/O（オーディオのインプットとアウトプット）を設定して最適化します。全部設定するのに10分ほどかかりますが、使用するインターフェースに対して一度設定してしまえば再び設定する必要はありません。

このレッスンを終えると、以下が完了されます。

-> ドライバタイプの選択
-> 入力と出力デバイスの選択
-> 入力と出力の設定
-> サンプリングレートの選択
-> 入力/出力バッファサイズの調節


下の「次のページ」ボタンを押して進みます。

$Page サンプルレートの選択
$TargetName Selecting the Sample Rate

「入力/出力サンプルレート」設定では、$0とオーディオインターフェースが動作するサンプリングレートを決定します。オーディオデバイスによっては、レートが固定されている場合もあります（通常44,100Hz）。
SampleRate.tif

$0はリアルタイムサンプルレート変換を行うため、$0ではサンプルレートの異なるオーディオクリップを使用することができます。しかし、新規オーディオクリップはすべて、ここで選択されたサンプルレートで録音されます。


Divider.tif
参考情報：「サンプルレート」とは、1秒間にデジタル変換されるサウンドのサンプル数を意味します。サンプルレートが高ければ高いほど、サウンドのデジタル変換がより性格になります。しかし、1秒当たりのオーディオ演算処理（サンプル処理）が増えるため、CPU負荷も高くなります。CDと同レベルの品質のオーディオサンプルレートは、1秒当たり44,100サンプルですので、CD録音と同じクオリティのオーディオを作成したい場合、サンプルレートを44,100に設定するとよいでしょう。

$Comment --------------------------------------------------------------------------------
$Page テストトーンを使う
$TargetName Using the Test Tone

コンピュータ内を通過するオーディオはすべて、「バッファ」に一時的に保管されます。「バッファ」とは、CPUが別のタスクを処理している間もオーディオストリームの流れを止めないようにするためのサウンドのため池のようなものです。バッファが大きいと、オーディオの流れが中断されませんが、ソフトウェアの反応がいくらかゆっくりになります。バッファが小さいと、ソフトウェアの反応は良くなりますが、サウンドがとぎれたり中断したりします。

「バッファサイズ」設定では、反応性と安定性の最適なバランスを見極めることができます。ご使用のシステムに適した値を見つけるには、$0をシミュレートし、複数のトラック、バーチャルインストゥルメント、エフェクトを使用した状態でどれくらいの負荷がCPUにかかるのか判断する必要があります。

$0の「オーディオ」設定の「テスト」セクションにある「トーンボリューム」パラメータを選択し、最低値の「-48dB」まで下げます。
ToneVolumeMin.tif

「CPU使用量シミュレータ」を最高値の「80%」まで上げます。
CPUUsageSim.tif

$Page 出力バッファサイズの調節
$TargetName Adjusting the Output Buffer Size

「テストトーン」スイッチをクリックし、テストトーンをオンにします。
TestTone.tif

テストトーンによりCPU使用量シミュレータがオンになるため、この時点で動作速度がいくらか低下することがあります。

テストトーンがはっきり聞こえるようになるまで、トーンボリュームをゆっくり上げていきます。それから、「レーテンシー」セクションの「出力バッファサイズ」コントロールを最低値に設定します。
OutputBufferSizeControlDirectXMme.tif

テストトーンの出力にクリックノイズやドロップアウトが聞こえます。クリックノイズやドロップアウトが聞こえなくなるまで、バッファサイズをゆっくり上げていきます。信号が問題なく通過する値を見つけましょう。マウスクリックでスライダーを選択してから、キーボードの矢印キーを使って調整することもできます。

きちんと設定できたら、テストトーンをオフに切り替えます。


Divider.tif
ヒント：ハードディスク動作により、最小バッファサイズに制約が加わることがあります。オーディオトラックを複数再生・録音する際にサウンドが中断するようであれば、バッファサイズを少し上げる必要があります。

スイッチの右の列ではステレオとして使用するチャンネルをオンにし、左の列では個別にモノとして使用するチャンネルをオンにすることができます。

例えば、インプット1と2が常にターンテーブルに接続されている場合、ステレオで使用します。一方、インプット1と2が常にマイクに接続されている場合、モノスイッチをオンにする必要があります。

両方のスイッチをオンにして、場合に応じて最適な設定を選択することも可能です。この設定は、接続がひんぱんに変更されるような場合に便利です。たとえば、コンピュータ内蔵のステレオハードウェアインターフェースを使用する場合、マイクからモノで録音し、接続を変更してから、ターンテーブルからステレオで録音します。

使用しないインプットまたはアウトプットに対しては、「両方」のスイッチをオフにします。未使用のチャンネルにたいしてオフに設定することで、CPUのパフォーマンスを最適にすることができます。
