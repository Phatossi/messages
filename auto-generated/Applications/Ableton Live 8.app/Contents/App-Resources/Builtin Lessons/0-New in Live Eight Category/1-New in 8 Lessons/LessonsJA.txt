$Page 新ワーピングエンジン
$TargetName New Warping Engine

Liveの「ワーピング」機能では、トランジェント分析が向上し、インターフェースに改良が加えられています。 最も注目したいのは、ワープマーカーの作成と変更がさらに直感的に行えるようになった点です。 ワープマーカーをサンプル内の特定の位置に置いてから、固定のタイムラインに沿ってドラッグすることができるようになりました。
WarpMarkers.tif

ズームインすると、トランジェントが小さなマーカーとして表示され、さらに編集を加えることができます。
Transients.tif

トランジェント条をマウスオーバーすると、「疑似ワープマーカー」が表示されます。 これをダブルクリックまたはドラッグして、本物のワープマーカーを作成することができます。
PseudoWarpMarker.tif

ビートモードの新しい「トランジェントループ」と「エンベロープ」コントロールにより、特にゆっくり目のテンポの場合のワーピングのクオリティが向上しています。
TransientsLoopAndEnvelope.tif

また、[編集]メニューの[クオンタイズ]コマンドを使って、オーディオクリップを「クオンタイズ」することができるようになりました。
QuantizeAudio.tif

$Page グルーヴ新機能
$TargetName New Groove Functionality

Liveの新しい「グルーヴエンジン」により、微妙（または極端）なタイミング動作を、クリップ単位でダイナミックかつリアルタイムにコントロールすることができるようになりました。 グルーヴは、グルーヴライブラリからドラッグし、セット内のオーディオクリップまたはMIDIクリップへドロップすることができます。
GrooveLibrary.tif

セットで使ったグルーヴは、どれも「グルーヴプール」に表示され、そこでさらに変更を加えることができます。
GroovePool.tif

さらに、コンテクストメニューのコマンドを使って、オーディオクリップまたはMIDIクリップからグルーヴを「抽出」することもできます。コンテクストメニューは、クリップを $ContextMenuing すると選択できます。
ExtractGroove.tif

グルーヴファイルの内容は、ブラウザまたはグルーヴプールからセットへファイルをドラッグして表示することができます。セットでは、グルーヴファイルはMIDIクリップへと変換されます。

アレンジメントビューで、隣り合うクリップ間での「クロスフェード」を作成することができるようになりました。

$Page MIDIエディタの機能向上
$TargetName MIDI Editor Enhancements

MIDIエディタが刷新され、MIDI編集がアレンジメント内での編集と同じように行えるようになりました。

「挿入マーク」で正確な位置を定め、クリップ内の至るところでノートまたはタイムをコピー・ペースト・挿入・削除することができます。
InsertMark.tif

機能強化されたキーボードコマンドにより、編集ワークフローも向上しています。 全コマンドの一覧について、詳しくはリファレンスマニュアルの「MIDIノートとベロシティの編集」の章をご参照ください。

挿入マークにより、MIDIノートの「ステップ録音」も可能になります。
StepRecording.tif
まずMIDIトラックをアーム（録音可能に）し、クリップのプレビュースイッチをオンにし、MIDIエディタをクリックしてから、録音を開始したい位置に挿入マークを置きます。 これで、右矢印キーを押しながらノートを押し続けると、ノートがクリップに追加され、挿入マークは次のグリッド位置へ進みます。

$Page グループトラック
$TargetName Group Tracks

オーディオトラックとMIDIトラックは、 「グループトラック」としてひとつにまとめることができます。
GroupTrackSession.tif

グループトラックは、表示を切り替えセッションビューとアレンジメントビューの両方でワークフローの整理に活用できます。
GroupTrackArrangement.tif

グループトラックにクリップを入れることはできませんが、ミキサーコントロールがあり、オーディオエフェクトをホストすることができる点で、オーディオトラックに似ています。

グループトラックのスロットのシェーディング部分は、グループに含まれている1つ以上のトラックのそのシーンにクリップがあることを示しています。 シェーディングの色は、シーン内の一番左のクリップの色です。
GroupSlots.tif

これらのグループスロットにもラウンチボタンが付いており、グループのクリップすべてをこのボタンでラウンチできます。 対応するクリップがないグループスロットには停止ボタンが付いています。 グループスロットをクリックすると、関連するすべてのクリップが選択されます。

$Page 複数のパラメータを同時調整
$TargetName Multiple Parameter Adjustment

複数のトラックが選択されている場合、そのうちの1つのミキサーやパラメータを調整すると、選択されている他のトラックのパラメータを同時に調整することができます。
TrackGroups.tif

複数選択されているクリップの特定のノブやスライダーのパラメータ（ボリュームなど）がクリップごとに異なる値に設定されている場合、パラメータを調節すると、値の差異が維持されます。

同時に調整可能なミキサーのパラメータには以下があります。

-> ボリューム
-> パン
-> センド
-> トラックアクティベータ、ソロボタン、アームボタン
-> ルーティングセレクタとモニタースイッチを含む入力/出力セクション
-> アレンジメントビューのオートメーションセレクタとトラックの展開ボタン

また、セッションビューミキサーで複数のチェーンが選択されている場合、チェーンのプロパティも同時に調整することができます。

$Page Collision と Corpus
$TargetName Collision & Corpus

Collisionは、マレットを使う打楽器の特性をシミュレートするシンセサイザです。 Applied Acoustics Systems（AAS）とのコラボレーションにより開発されたCollisionは、フィジカル・モデリング技術を使用し、実在する物体（または実在しない物体）のサウンド生成と共鳴装置をモデリングしています。
Collision.tif

Corpusは、Collisionの共鳴装置セクションを再現した別個のエフェクトです、Corpusでは、フィジカルモデルを使ってオーディオソースを処理することができます。 Corpusは、独自のMIDIサイドチェーン機能も搭載しています。この機能により、MIDI入力を介してデバイスの共鳴装置をチューニングすることができます。
Corpus.tif

$Page Limiter
$TargetName Limiter

LiveのLimiterは、マスタリング品質のダイナミックレンジ・プロセッサで、オーディオレベルがあらかじめ設定したスレッショルドを超えないよう制御します。 これは、クリッピングを防ぐための安全装置として、またはサウンドデザインのツールとして使用することができます。
Limiter.tif

[Ceiling]スライダーでは、デバイス出力の絶対最大値を設定します。また、[Gain]ノブでは、リミッティング前に入力信号をブーストまたは減衰することができます。

手動または自動のリリースタイムでは、信号レベルが上限を下回る時、Limiterがどれくらいすばやく処理を停止するかを調整します。

[Stereo]スイッチでは、ピークをチャンネルごとに個別に制限したり、どちらか1つのチャンネルが上弦を超えた時に両チャンネルを制限することができます。

$Page Multiband Dynamics
$TargetName Multiband Dynamics

Multiband Dynamicsデバイスは、オーディオ素材のダイナミックレンジを変更するフレキシブルなマスタリング・ツールです。
MultibandDynamics.tif

Multiband Dynamicsでは、3つの周波数帯域のコンプレッションとエクスパンションが行えます。各周波数帯域に対しては、それぞれクロスオーバー・ポイントとエンベロープ・コントロールを調整できます。 各周波数帯域には上側スレッショルドと下側のスレッショルドの両方があり、帯域ごとに2種類のダイナミクス処理を同時に使用することができます。

直感的な操作が可能なディスプレイ・コントロールを上下にドラッグして、特定のダイナミックレンジ内でボリュームを調整することができます。 小さな黄色のバーは入力レベルを、大きなオレンジ色のバーは出力を示しています。

一般的なマルチバンド・コンプレッションとして使用するほか、ディエッシングやノイズ・リダクション、さらにはアンコンプレッションなどの少し変わったマスタリング・タスクにも使用できます。

$Page Overdrive
$TargetName Overdrive

Overdriveは、Liveのディストーション・ツールに追加されるパワフルな新エフェクトです。 ギターペダルに見られる回路に基づく、温かみのあるサチュレーションを加えます。

Overdrive.tif

低いレベルでは、Overdriveは優しい温かみを加えるエフェクトとして使用できます。 また、大きなディストーションを加えたり、トーンシェイピングに使用することもできます。

[Dynamics]スライダーでは、このエフェクトにより適用される内部コンプレッションとメイクアップゲインの量をコントロールすることができます。

$Comment --------------------------------------------------------------------------------
$Page Frequency Shifter
$TargetName Frequency Shifter

Frequency Shifterエフェクトは、周波数偏移とリングモジュレーション・エフェクトが1つのパワフルなデバイスに組み合わせられています。
FrequencyShifter.tif

周波数シフトでは、ユーザがヘルツ単位で定義する周波数を入力信号に加減します。リングモジュレーションでは、このオフセット値を同時に加減します。 少しだけ変化させると、微妙なトレモロやフェージングの効果を得ることができます。大きくずらすと、不協和でメタリックなサウンドを作成することができます。

周波数シフトは、内部LFOでも調節することができます。

$Page Enhanced Plug-In Support
$TargetName Enhanced Plug-In Support

サードパーティ製プラグインの編集は、最初の128パラメータに限らず行えるようになりました。 トラックビューのプラグインには「設定モード」が加えられました。設定モードでは、プラグインウィンドウのデバイスのパラメータをクリックしてパネルに加えることができます。
ConfigureMode.tif

パラメータの作成が完了したら、これを記録し、必要に応じて入れ替えたり削除したりできます。

パラメータは、プラグインウィンドウ内のパラメータ変更をオートメーションとして記録したり、MIDI、キー、マクロの各マップモードでパラメータをクリックすることで、パネル内に作成することもできます。

設定モードを終了したら、他のLiveの内蔵デバイスと同じように、パネル内のパラメータを変更してMIDIコントローラ・マクロ・キーボードへマップすることができます。

編集ウィンドウを持たないプラグイン、またはパラメータ数が32未満のプラグインは、Liveのパネル内にすべてのパラメータが表示されます。これらのパラメータも、動かしたり並べ替えたりすることができます。

$Page その他の機能向上
$TargetName Additional Small Improvements

Live 8では、多数の機能およびワークフローの向上が行われています。

-> マクロコントロールは、単一のパラメータにアサインされている場合、そのパラメータ名とユニットを自動的に借用します。
-> トランジェントに基づいてオーディオファイルをスライスすることが可能になりました。
-> インストゥルメントとエフェクトのデバイスを、標準のクリップボードコマンドを使ってコピー・ペースト・複製できるようになりました。
-> 隣り合わない複数のトラックとシーンを選択・コピー・移動できるようになりました。
-> Flangerエフェクトに「ハイクオリティモード」が追加されました。
-> MIDIトラック内のインストゥルメントは、トラックが録音可能である場合コントロールサーフェスに自動的にロックされます。
-> トラックとチェーンのボリュームは小数点第3位まで表示されるようになり、より正確になりました。 値はステータスバーに表示されます。
-> ファイルマネージャに、各参照サンプルが実際に使用されている現在のLiveセット内のクリップとインストゥルメントすべてがリストされるようになりました。
-> Ping Pong DelayとSimple Delayにユーザアサイン可能な「ディレイトランジションモード」が追加されました。ディレイトランジションモードは、デバイスのタイトルバーを $ContextMenuing して選択できます。
-> Samplerのウェーブシェイパーに4つ目のカーブタイプ[4bit]が追加されました。
-> Sampler にモーフィング可能な SVF フィルタが追加されました。
-> 複数の「Liveパック」を同時にアンインストールできるようになりました。
