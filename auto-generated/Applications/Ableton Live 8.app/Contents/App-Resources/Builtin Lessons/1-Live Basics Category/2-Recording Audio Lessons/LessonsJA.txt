$Page アレンジメント
$TargetName The Arrangement

このレッスンでは、$0の「アレンジメントビュー」での操作について学びます。アレンジメントビューは、現在アプリケーション・ウィンドウの中上部に表示されています。
OverviewArrangement.tif

アレンジメントは、テープレコーダーと同じように使用することができます。アレンジメント内には、サウンドを自由に録音することができます。録音したサウンドは、好きなときに、好きな位置から再生することができます。作品が録音できたら、$0からエクスポートし、CDまたはDVDに保存することができます。

スクリーン左側上の「ブラウザ」を隠し、アレンジメントの表示スペースを広げてみましょう。インフォビュー同様、ブラウザにも展開ボタンが付いています。
BrowserShowHideRec.tif

$Page 入力ルーティング
$TargetName Input Routing

$0ミキサーの入力/出力セクションを使って、各トラックにおけるオーディオ信号の発信元（入力）とその送信先（出力）を指定します。
ArrangementIOSection.tif

入力/出力セクションの「入力タイプ」セレクタの表示を見ると、外部チャンネル1および2からオーディオがトラックに発信されていることが分かります。ほとんどの場合、コンピュータの標準オーディオ入力（内蔵マイクまたはラインイン・ジャック）となっています。
Input1And2Rec.tif

上の入力セレクタでは、外部入力（'Ext.In'）、他のトラック、ReWireアプリケーションからの信号などのさまざまな「入力ソース」から選択できます。

下の入力セレクタでは、さまざまな「入力チャンネル」を選択できます。マルチチャンネル・オーディオ・インターフェースを使って作業する場合にしばしば使用されます。


Divider.tif

$Comment --------------------------------------------------------------------------------
$Page オーディオトラックをアームする
$TargetName Arming an Audio Track

録音を始める前に、この下に表示されているオーディオ・トラックの「アーム」ボタンをクリックし、トラックを「アーム」する必要があります。こうすると、$0は録音処理を行うことができるようになります。

メモ：トラックをアームする際にフィードバックが聞こえる場合、アームボタンをもう一度押してトラックを無音にし、状況を確認してみてください。
Track1ArmedRec.tif

アームされると、トラックの「レベルメーター」にトラック入力の信号レベルが表示されます。これは、録音されるサウンドのボリュームを示しています。
InputMeterRec.tif

メーターに何の信号も表示されない場合は、まず、突然のボリュームの変化により耳または装置に損傷をきたすことのないよう、スピーカーまたはヘッドフォンのボリュームを下げます。そして、インストゥルメントまたはマイクが接続されている入力チャンネルを選択しているかどうかを確認します。問題を突き止め解決できたら、ボリュームを適切なレベルへ戻しましょう。

$Comment --------------------------------------------------------------------------------
$Page モニタリング
$TargetName Monitoring

「モニタリング」では、選択したトラック入力の信号が録音中にどのように聞こえるかを設定します。

入力/出力セクションの3つのスイッチを使って、各オーディオおよびMIDIトラックのモニタリング動作をそれぞれ設定します。
MonitoringSectionRec.tif

-> 「直接」モニタリングする場合、「Off」を選択します。「直接モニタリング」とは、オーディオインターフェースまたは外部ミキシングボードを通じて、あるいは、生でモニタリングを行うことをいいます。

-> $0を介して入力信号をモニタリングする場合、「Auto」を選択します。共鳴音が聞こえたりフィードバックが発生する場合は、「直接モニタリング」に設定されています。トラックのモニタリング・スイッチを「Off」に設定するか、ご使用のオーディオ・インターフェースの直接モニタリング・オプションをオフに切り替えます。

ソフトウェアを介してモニタリングしている場合は、遅延または「レーテンシー」が生じ、動作から少し遅れて音が聞こえます。レーテンシーを最小限にする方法について、詳しくは $Link オーディオI/Oを設定する <Lesson:Builtin/3-Hardware Setup Category/1-Setting up Audio IO>レッスンをご参照ください。

$Page 録音準備を行う
$TargetName Preparing to Record

$0内に現れたオーディオ信号をモニタリングできたら、アレンジメント内にテイクを録音できます。まず、オーディオトラックがアームされていることを確認します。
Track1ArmedRec.tif

コントロールバーの「停止」ボタンを2度押します。アレンジメントが1.1.1.（開始位置）へリセットされます。
TransportStopRec.tif

次に、となりの「グローバル録音」ボタンを押します。（マルチトラックテープレコーダを使ったことがあれば、同様の操作なので分かりやすいかもしれません。各トラックをアームし録音可能にしていれば、グローバル録音ボタンを使ってそれらのトラックを録音可能に切り替えることができます。）
GlobalRecordRec.tif

最後に、コントロールバーの「再生」ボタンを押し（または、ご使用のコンピュータ・キーボードの「スペースキー」を押し）、録音を開始します。
TransportPlayRec.tif

さあ、音を鳴らしてみてください。

$Page アレンジメントのナビゲーション
$TargetName Navigating the Arrangement

録音が一定の長さを超えると、アレンジメントが自動的に水平にスクロールします。表示部分を前後にスクロールしたり、ズームイン・ズームアウトを行うには、ビートタイムルーラーを上下左右にドラッグします。
-> ズームインまたはズームアウトするには、マウスを垂直に動かします。
-> 左右にスクロールするには、マウスを水平に動かします。
TimeLineRec.tif

再生と録音は、ソングのどの位置からも行えます。(クリップのタイトルバー以外の部分の)アレンジメントビューをクリックして｢挿入マーカー｣を設定すると、ソングがその位置から再生を開始するようになります。
InsertMarker.tif

録音した内容に満足がいかない場合も大丈夫。$CommandMod+Zを押すだけでテイクを取り消し、もう一度やり直すことができます。

$Page メトロノーム
$TargetName The Metronome

すべてが同期で再生されるようにするためには、同期で録音することが非常に重要になります。

同期で録音する最も簡単な方法は、$0内蔵の「メトロノーム」を使用することです。メトロノームは、コントロールバー・スイッチからオンにできます。スイッチは、アプリケーション・ウィンドウ左上隅にあります。
MetronomeSwitch.tif

$0が再生中、メトロノームは現在のテンポを刻みます。メトロノームをオンにしてからスペースキーを押し、再生を開始します。マスタートラックの「キュー・ボリューム」コントロール（下図）を使って、メトロノームのボリュームを調節することができます。停止するには、スペースキーをもう一度押します。
CueVolume.tif


Divider.tif
メモ：録音内容がメトロノームに比べて遅れて聞こえるようであれば、「レーテンシー補正」を設定する必要があります。レーテンシー補正とは、直接モニタリングの場合にプログラムが行う、オーディオ・インターフェースとそのドライバにより生じる遅延を補正するための操作です。この補正を設定するには、$0を使ってテストを行う必要があります。テストの手順について、詳しくは $Link ドライバエラー補正 <Lesson:Builtin/3-Hardware Setup Category/2-Driver Error Compensation>レッスンをご参照ください。

$Page テンポのテクニック・カウントイン・タッピング
$TargetName Tempo Tricks, Count-in and Tapping

Liveセットのテンポは、録音前・録音中・録音後と、いつでも変更できます。たとえば、複雑なギターソロの録音中はテンポを落としておき、録音後テンポを上げるということが可能です。
TempoField.tif

もうひとつの便利なテクニックに、録音中に「カウントイン」を使用する方法があります。この機能を使用すれば、テイク録音の開始前にカウントが入り、準備がしやすくなります。この動作を設定するには、オプションメニュー（PC）または$0メニュー（Mac）から「環境設定」を開き、「録音/ワープ/ラウンチ」タブを選択します。カウントインが「None」以外に設定されている場合、カウントインが完了するまで$0は録音を開始しません。
CountInPref.tif

また、あなたのカウントに続いて$0が再生を始めるよう設定することもできます。4分の4拍子の場合、[TAP]ボタンを4回クリックすると$0が再生（または録音）を開始します。
TapButton.tif

$Comment --------------------------------------------------------------------------------
$Page マルチトラック録音
$TargetName Multitrack Recording

$CommandMod+Tを押し、次のテイク用の新しいオーディオ・トラックを作成します。タイトルバーの展開ボタンをクリックし、新しいトラックを展開します。
NewTrackArm.tif

新しいトラックのアームボタンをオンにすることを忘れないでください。さもないと、またトラック1に録音されてしまいます。デフォルトでは、1つのトラックをアームすると、他のすべてのトラックのアームが解除されます。しかし、トラックのアームボタンを$CommandMod-クリックすることで、アーム状態を個別に設定することができます。こうして、複数のトラックで同時に録音を行うよう設定することができます。
MultiArm.tif

異なる入力信号（バンドの各メンバーからの信号など）をトラックが受信するよう設定されていなければ、このように設定する意味がありません。上図の例では、入力が変更されており、トラック1は入力1をモノで録音し、トラック2は入力2をモノで録音するようになっています。

テンポの目安にメトロノームを使用し、新しいテイクを録音しましょう。終わったら、停止ボタンかスペースキーを押して録音を停止します。

$Page テイクを重ねる
$TargetName Doing it Again (and Again...)

テイクの一部分だけを取り消したい場合は、「パンチ位置」を設定することができます。「ループブレース」の左右端をドラッグして、パンチイン・パンチアウト位置を設定します。
PunchMarkers.tif

... その後、コントロールバーの「パンチイン」と「パンチアウト」スイッチをオンにします。
PunchSwitches.tif

-> パンチインスイッチがオンの場合、ループブレースの前にあるマテリアルが保護されます。
-> パンチアウトスイッチがオンの場合、ループブレースの後にあるマテリアルが保護されます。

ループブレースを使って、アレンジメントのループ部分を決定することもできます。ループを設定して何度でもテイクを重ねることができ、$0には、ループするたびにそのオーディオが一時保存されます。ループ録音は、「取り消し」機能を使って、後から「ループを解除して展開する」ことができます。ループは、コントロールバーの「ループスイッチ」を使ってオンにします。
LoopSwitch.tif
